# 详细设计：音视频处理管线（占位）

## 目标

支持 **导入** → **时间线编排** → **预览** → **导出**；当前仓库初始化阶段仅建立 **扩展点** 与 UI 占位。

## 后续技术选项（未锁定）

- 浏览器内：**Web Audio**、**WebCodecs**、**ffmpeg.wasm** 等，按性能与格式支持评估。
- 桌面封装：Electron / Tauri 等，便于调用本地 FFmpeg。

## 与 UI 的边界

- 时间线与客户区仅通过 **明确的 TypeScript 接口**（见物理层 `spec.md`）接收「片段列表、播放头」等；具体解码实现在后续迭代填充。
